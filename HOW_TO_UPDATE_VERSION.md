# 🔄 คู่มือการอัปเดตเวอร์ชันโปรแกรม SwapTxT (How to Update Version)

การอัปเดตเวอร์ชันโปรแกรม (เช่น จาก 1.0.1 ไปเป็น 1.0.2) มีขั้นตอนการเปลี่ยนเลขเวอร์ชันในไฟล์ต่างๆ เพื่อให้ระบบโปรแกรม, ข้อมูลโปรเจกต์, และระบบ **Auto-Update** ทำงานสอดคล้องกันทั้งหมด โดยมีขั้นตอนดังต่อไปนี้:

## 📝 ส่วนที่ 1: การแก้ไขซอร์สโค้ด (Source Code)
เริ่มจากการบอกตัวโปรแกรม .NET ว่าตอนนี้เป็นเวอร์ชันอะไร:

**1. แก้ไขไฟล์ `Models\AppSettings.cs`**
ค้นหาบรรทัดที่มีข้อความ `public static string AppVersion` (ประมาณบรรทัดที่ 45) แล้วเปลี่ยนเป็นเลขเวอร์ชันใหม่ล่าสุด:
```csharp
[JsonIgnore]
public static string AppVersion => "1.0.2";
```
*(ส่วนนี้จะไปอัปเดต UI ในหน้าต่างตั้งค่า และหน้า About)*

**2. แก้ไขไฟล์ `SwapTxT.csproj`**
ค้นหาบล็อก `<PropertyGroup>` แล้วเปลี่ยนแท็ก `<Version>` ให้เป็นเลขเวอร์ชันใหม่ (แบบ 4 หลัก):
```xml
<Version>1.0.2.0</Version>
```

---

## 📡 ส่วนที่ 2: การแก้ไขระบบ Auto Update
เพื่อให้ผู้ใช้ที่เปิดโปรแกรมอยู่ได้รับการแจ้งเตือนอัตโนมัติ คุณต้องแก้ไฟล์เหล่านี้:

**3. แก้ไขไฟล์ `update.xml`**
ไฟล์นี้คือไฟล์ที่ระบบเช็กอัปเดตจะเข้ามาอ่านจาก GitHub ให้เปลี่ยน `<version>` เป็นเวอร์ชันล่าสุด:
```xml
<version>1.0.2</version>
```

**4. แก้ไขไฟล์ `version.json`** (ทางเลือกเสริม)
แก้ไขชื่อและเวอร์ชันให้เป็นปัจจุบัน:
```json
{
  "version": "1.0.2",
  "name": "SwapTxT Release 1.0.2",
  "date": "2026-04-20"
}
```

---

## 📄 ส่วนที่ 3: การแก้ไขคู่มือและแจกจ่าย
เตรียมคู่มือและไฟล์ให้ผู้ใช้:

**5. แก้ไขชื่อและการอ้างอิงใน `README.md`**
เปลี่ยนชื่อไฟล์ที่ให้คนดาวน์โหลดให้ตรงกับเวอร์ชันปัจจุบัน (เช่น เปลี่ยนจาก `V1-0-1` เป็น `V1-0-2`)

**6. สั่ง Build ให้เป็นไฟล์เดียวจบ**
รันคำสั่ง .NET เพื่อแพ็กไฟล์ทั้งหมดเป็น `.exe` อันเดียวไม่มีไฟล์อื่นเกะกะ:
```powershell
dotnet publish SwapTxT\SwapTxT.csproj -c Release -r win-x64 --self-contained true /p:PublishSingleFile=true /p:PublishReadyToRun=true /p:IncludeNativeLibrariesForSelfExtract=true -o Build_Release
```

---

## 🚀 ส่วนที่ 4: การนำขึ้น GitHub (Deployment)

1. กลับไปที่โปรเจกต์ (Repository) ใน GitHub ขิงคุณ 
2. ดำเนินการ **Commit** และ **Push** โค้ดทั้งหมดที่แก้ไขขึ้นไป (รวมถึงไฟล์ `update.xml`)
3. เข้าไปที่เมนู **Releases** ด้านขวามือของ GitHub
4. สร้าง **Draft a new release**
    - ระบุ Tag: `v1.0.2`
    - ตั้งชื่อ Title: `SwapTxT - Official Release v1.0.2`
5. นำไฟล์ `SwapTxT.exe` ในโฟลเดอร์ `Build_Release` ไปแจกจ่าย หรือลากไปใส่ในช่อง Assets แล้วกด **Publish release**

> **ข้อมูลสำคัญ:** ทันทีที่คุณกด Push ไฟล์ `update.xml` ขึ้น GitHub ไปปุ๊บ ระบบผู้ใช้งานเก่าทั่วโลกจะพบอัปเดตบนหน้าจอทันที!
